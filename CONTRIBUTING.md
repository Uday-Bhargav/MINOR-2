# Contributing to Global Crisis Market Intelligence Platform

Thank you for your interest in contributing to the project! Please follow these guidelines:

## Branch Naming Convention

Please use descriptive branch names starting with a category prefix:
- \eature/your-feature-name\ - For new features
- \ugfix/issue-description\ - For bug fixes
- \hotfix/urgent-fixes\ - For critical hotfixes
- \docs/update-documentation\ - For documentation updates
- \chore/cleanup\ - For routine tasks

## How to Submit Pull Requests

1. Fork the repository and create your branch from \main\.
2. Ensure you have tested your changes locally.
3. If you've added code that should be tested, add tests.
4. Ensure your code lints and is formatted correctly.
5. Create a Pull Request against the \main\ branch.
6. Provide a clear and detailed description of the changes in your PR.

## Code Style Guidelines

- **Python**: Follow PEP 8 guidelines. Use \lack\ for formatting and \lake8\ for linting.
- **JavaScript/React**: Use modern ES6+ features, functional components, and Hooks. Follow Prettier/ESLint defaults.
- Ensure all functions and complex logic are documented.
- Use explicit and meaningful variable/function names.
